# Golden Set Sync — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add two slash commands (`/improve-golden-set` and `/update-claude`) for bidirectional sync between the golden set repo and bootstrapped projects.

**Architecture:** Both commands are prompt-only markdown files in `golden/.claude/commands/`. They follow the same conventions as existing commands: YAML frontmatter, phased steps, rules section. No code, scripts, or tests — these are Claude instruction files.

**Tech Stack:** Markdown command files following the established golden set conventions.

**Design doc:** `docs/plans/2026-02-24-golden-set-sync-design.md`

### Task 1: Write `/improve-golden-set` command

**Files:**
- Create: `golden/.claude/commands/improve-golden-set.md`

**Step 1: Write the command file**

Create `golden/.claude/commands/improve-golden-set.md` with:

- YAML frontmatter: `name: improve-golden-set`, `description: Extract generalized improvements from a bootstrapped project back into the golden set`, `user_invocable: true`
- Title: `# /improve-golden-set — Golden Set Extraction`
- Overview paragraph explaining purpose: run from the golden set repo, point at a bootstrapped project, extract generalizable improvements

**Invocation section:**
```
/improve-golden-set ~/dev/my-project
```
If no path argument provided, ask the user for one. Do not proceed without a valid path.

**Phase 1: Validate & scan target project**

Validate the path:
- Directory exists
- Has `CLAUDE.md` with the bootstrap marker (`<!-- bootstrap-claude: project-specific below -->`)
- Has `.claude/` directory

Read the target project's Claude configuration:
- `CLAUDE.md` — split at bootstrap marker into baseline (above) and project-specific (below)
- `.claude/commands/*.md` — all command files
- `.claude/agents/*.md` — all agent files
- `.claude/settings.local.json`
- `.mcp.json`

Read the golden set equivalents from the current repo's `golden/` directory.

Classify each target item:
- **Golden original:** Content matches the golden set exactly
- **Golden modified:** Exists in golden set but has been edited in the project
- **Novel:** Exists in the project but not in the golden set

Skip these (never extract):
- Content below the bootstrap marker in CLAUDE.md
- `.claude/lessons.md`
- `tasks/todo.md`
- Project-specific commands that were generated by `/bootstrap-claude` (e.g., `add-endpoint.md`, `add-component.md`, `add-pipeline-step.md`, `update-docs.md`)

**Phase 2: Diff against golden set**

For each classified item, build a diff report:

- **CLAUDE.md baseline:** Compare the project's baseline sections (above marker) line-by-line against `golden/CLAUDE.md`. Flag any additions, removals, or modifications.
- **Commands:** For each golden-modified command, diff against `golden/.claude/commands/<name>.md`. For each novel command, summarize its purpose and content.
- **Agents:** Same as commands — diff modified, summarize novel.
- **settings.local.json:** Compare the `allow` arrays. Identify permissions in the project that aren't in the golden baseline. Filter out obviously project-specific permissions (language/tool-specific like `go build`, `cargo test`, `docker`, `psql`, `jira`, `glab`, `linear`). Flag remaining new permissions as potentially baseline-worthy.
- **.mcp.json:** Diff server configurations.

If nothing has changed or everything is project-specific, report "No extractable improvements found" and stop.

**Phase 3: Propose extractions**

Present findings to the user one at a time, grouped by type:

1. **CLAUDE.md baseline improvements** — show the diff, propose how to integrate into `golden/CLAUDE.md`
2. **Modified commands** — show the diff, propose whether the change is universal (update golden) or project-specific (skip)
3. **Novel commands** — show the command content, propose a generalized version that strips project-specific references (file paths, framework names, etc.) and replaces them with generic placeholders or patterns. Recommend: include in golden set, or skip.
4. **Novel agents** — same as novel commands
5. **New baseline permissions** — list them, recommend include or skip
6. **MCP server changes** — show diffs, recommend include or skip

For each item, present:
- **What:** The specific content or diff
- **Generalization:** How it would look as a universal golden set item
- **Recommendation:** Include / Skip / Needs manual editing

Wait for user to approve, reject, or modify each item before moving to the next.

**Phase 4: Apply approved changes**

For each approved item:
- Update the corresponding file in `golden/` (e.g., `golden/CLAUDE.md`, `golden/.claude/commands/<name>.md`)
- For novel commands/agents, create the new file in `golden/.claude/commands/` or `golden/.claude/agents/`
- For CLAUDE.md changes, edit the golden CLAUDE.md directly

Do NOT commit. Present a summary of all files changed and let the user review and commit manually.

**Rules section:**
- NEVER extract content from below the bootstrap marker
- NEVER extract `.claude/lessons.md` content
- NEVER extract project-specific permissions (language/tool-specific)
- NEVER auto-commit — always leave changes for user review
- ALWAYS present each extraction for approval before applying
- ALWAYS generalize before extracting — strip project-specific references
- If a command references project-specific file paths, framework APIs, or tool names, generalize them or skip the command
- If the golden set already has a similar concept, propose updating the existing item rather than adding a duplicate

**Step 2: Review the command for consistency**

Read the completed file. Verify:
- Frontmatter matches the pattern in other commands (name, description, user_invocable)
- Phase structure is clear and sequential
- Rules section covers safety constraints
- Tone and style match existing commands (e.g., `/close-issue`, `/triage`)

**Step 3: Commit**

```bash
git add golden/.claude/commands/improve-golden-set.md
git commit -m "feat: add /improve-golden-set command for extracting improvements from projects"
```

### Task 2: Write `/update-claude` command

**Files:**
- Create: `golden/.claude/commands/update-claude.md`

**Step 1: Write the command file**

Create `golden/.claude/commands/update-claude.md` with:

- YAML frontmatter: `name: update-claude`, `description: Pull golden set updates into a bootstrapped project while preserving project-specific customizations`, `user_invocable: true`
- Title: `# /update-claude — Golden Set Update`
- Overview paragraph explaining purpose: run from a bootstrapped project, point at the golden set repo, pull in new/changed content

**Invocation section:**
```
/update-claude ~/dev/shared/dotfiles
```
If no path argument provided, ask the user for one. Do not proceed without a valid path.

**Phase 1: Validate**

Validate the golden set path:
- Directory exists
- Has `golden/` subdirectory
- `golden/CLAUDE.md` exists
- `golden/.claude/commands/` exists

Validate the current project:
- Has `CLAUDE.md`
- CLAUDE.md contains the bootstrap marker (`<!-- bootstrap-claude: project-specific below -->`)
- Has `.claude/` directory

If either validation fails, report the specific problem and stop.

**Phase 2: Diff golden set against project**

Read all golden set files and their project counterparts. For each, classify:

**CLAUDE.md baseline:**
- Extract the project's baseline (everything above the bootstrap marker)
- Compare against `golden/CLAUDE.md`
- Classify as: **identical**, **golden updated** (golden has changes the project doesn't), **project modified** (project changed the baseline), or **diverged** (both changed)

**Commands** (`golden/.claude/commands/*.md` vs `.claude/commands/*.md`):
- For each golden command, check if it exists in the project
  - **New:** In golden but not in project
  - **Unchanged:** Identical in both
  - **Golden updated:** Golden changed, project has old version
  - **Project modified:** Project changed, golden has old version
  - **Diverged:** Both sides differ from each other
- Ignore project commands that don't exist in golden (those are project-specific, leave them alone)

**Agents** (`golden/.claude/agents/*.md` vs `.claude/agents/*.md`):
- Same classification as commands
- For agents with bootstrap markers (e.g., code-reviewer.md), only compare content above the marker

**settings.local.json:**
- Parse the `allow` arrays from both
- Identify permissions in golden that aren't in the project (new baseline permissions)
- Never flag project-specific permissions for removal

**.mcp.json:**
- Compare server configurations
- Identify new servers in golden

If everything is identical, report "Already up to date" and stop.

**Phase 3: Present changes for approval**

Group findings by type and present:

**New items** (in golden, not in project):
- List each with a summary
- Default recommendation: add
- Example: "New command `/improve-golden-set` — extract improvements from projects. Add to this project?"

**Updated items** (golden changed, project has old version):
- Show the diff between versions
- Default recommendation: update
- Example: "Command `/triage` updated in golden set. Changes: [diff]. Update?"

**Diverged items** (both sides changed):
- Show both versions side by side
- Ask user to choose: accept golden version, keep project version, or describe a merge
- Example: "Command `/close-issue` differs in both golden and this project. Golden version: [content]. Project version: [content]. Which to keep?"

**Removed from golden** (project has a golden command that golden no longer includes):
- Flag for user awareness
- Default recommendation: remove
- Example: "Command `/old-command` was removed from golden set but still exists here. Remove?"

User approves/rejects each change individually.

**Phase 4: Apply approved changes**

For each approved change:
- **New commands/agents:** Copy from golden to project's `.claude/commands/` or `.claude/agents/`
- **Updated commands/agents:** Replace the project's version with golden's version
- **CLAUDE.md baseline updates:** Replace the project's baseline section (above marker) with golden's version. Preserve everything below the marker untouched.
- **New permissions:** Merge into project's `settings.local.json` `allow` array
- **MCP changes:** Update `.mcp.json`
- **Diverged items (golden chosen):** Replace with golden version
- **Diverged items (project chosen):** No change
- **Removed items:** Delete the file

Present a summary:
```
## Update Complete

### Changes Applied:
- CLAUDE.md baseline: Updated (N sections changed)
- Commands added: /command-a, /command-b
- Commands updated: /command-c
- Agents updated: code-reviewer.md
- Permissions added: N new baseline permissions
- [etc.]

### Unchanged:
- Project-specific CLAUDE.md sections: preserved
- Project-specific commands: untouched
- Project-specific permissions: preserved
```

**Rules section:**
- NEVER modify content below the bootstrap marker in CLAUDE.md
- NEVER remove project-specific permissions from settings.local.json
- NEVER remove project-specific commands (those not in golden set)
- NEVER auto-apply changes — always present each change for user approval
- NEVER touch `.claude/lessons.md`, `tasks/todo.md`, or `.claude/settings.json` (hooks)
- ALWAYS preserve project-specific agent augmentations (content below agent bootstrap markers)
- If the project's CLAUDE.md baseline diverged from golden, show both versions and let the user decide
- If a command was both modified locally and updated in golden, treat as diverged — never silently overwrite

**Step 2: Review the command for consistency**

Read the completed file. Verify:
- Frontmatter matches the pattern in other commands
- Phase structure is clear and sequential
- Rules section covers all safety constraints
- Tone and style match existing commands
- The "never touch below marker" constraint is clearly stated in multiple places

**Step 3: Commit**

```bash
git add golden/.claude/commands/update-claude.md
git commit -m "feat: add /update-claude command for pulling golden set updates into projects"
```
