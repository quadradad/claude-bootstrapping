# Golden Set Sync — Design

Two new slash commands for bidirectional sync between the golden set repo and bootstrapped projects.

## Problem

The golden set deploys to projects via `deploy.sh` + `/bootstrap-claude`. But there's no way to:
1. Extract improvements made in a project back into the golden set
2. Pull golden set updates into an already-bootstrapped project

## Solution

Two new commands in `golden/.claude/commands/`:

- **`/improve-golden-set <path>`** — Extract generalized improvements from a project back into the golden set
- **`/update-claude <path>`** — Pull golden set changes into a bootstrapped project

Both require a path argument. If none provided, ask for one.

---

## `/improve-golden-set <project-path>`

Run from the golden set repo. Analyzes a bootstrapped project for novel or improved Claude configuration and proposes generalized extractions.

### Phase 1: Scan target project

Read the target project's Claude configuration:
- `CLAUDE.md` — separate baseline (above bootstrap marker) from project-specific (below)
- `.claude/commands/` — all command files
- `.claude/agents/` — all agent files
- `.claude/settings.local.json` — permissions
- `.mcp.json` — MCP servers

Classify each item:
- **Golden original:** Unchanged from golden set
- **Golden modified:** Golden command/content that was edited in the project
- **Novel:** New content that doesn't exist in the golden set

### Phase 2: Diff against golden set

Compare baseline CLAUDE.md sections against `golden/CLAUDE.md`. Flag differences.
Compare each golden-origin command against its golden counterpart. Flag modifications.
List all novel commands/agents with summaries.
Compare settings.local.json for new baseline-worthy permissions.
Compare `.mcp.json` for new MCP servers.

### Phase 3: Propose extractions

For each finding, present to user:
- **What changed:** The specific diff or new content
- **Generalization:** How to express it as a universal concept (strip project-specific references)
- **Recommendation:** Include in golden set, skip, or needs manual editing

Wait for user approval on each item.

### Phase 4: Apply approved changes

Update `golden/CLAUDE.md`, `golden/.claude/commands/`, etc. with approved generalizations.
Do NOT commit — leave changes for user review.

### Exclusions (never extract)

- Content below the bootstrap marker in CLAUDE.md
- Project-specific commands generated by bootstrap (e.g., `/add-endpoint`, `/add-component`)
- Project-specific permissions (e.g., `docker`, `psql` added for one project)
- `.claude/lessons.md` content
- `tasks/todo.md`

---

## `/update-claude <golden-set-path>`

Run from a bootstrapped project. Pulls new/changed content from the golden set while preserving project-specific customizations.

### Phase 1: Locate and validate

Validate the provided path has `golden/` directory with expected structure.
Validate the current project has been bootstrapped (bootstrap marker exists in CLAUDE.md).

### Phase 2: Diff golden set against project

Compare each golden set file against its project counterpart:

- **CLAUDE.md baseline sections:** Diff `golden/CLAUDE.md` against project's baseline (above marker only). Never touch project-specific content below marker.
- **Commands:** Classify each as **new** (not in project), **unchanged** (identical), or **diverged** (both sides differ).
- **Agents:** Same classification.
- **settings.local.json:** Identify new baseline permissions. Never remove project-specific permissions.
- **.mcp.json:** Identify new MCP server configs.

### Phase 3: Present changes for approval

Group by type:
- **New items** — exist in golden, not in project. Default: add.
- **Updated items** — golden changed, project has old version. Show diff, recommend update.
- **Diverged items** — both sides changed. Show both versions, ask user to choose or merge.
- **Removed from golden** — project has it, golden dropped it. Flag for awareness, recommend removal.

User approves/rejects each change individually.

### Phase 4: Apply approved changes

Apply approved additions and updates.
For CLAUDE.md, only modify baseline sections — never touch below bootstrap marker.
For diverged items user chose to merge, apply the merge.
Present summary of what changed.

---

## Where commands live

Both commands live in `golden/.claude/commands/` and deploy automatically via `deploy.sh`'s rsync of the `.claude/` directory.

## What doesn't change

- `deploy.sh` — already copies entire `.claude/` tree, no changes needed
- `/bootstrap-claude` — no changes needed
- Existing commands — unaffected
